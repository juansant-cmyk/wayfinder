# Definition of Done

A user story is **Done** when all of the following are true:

## Product
- [ ] Acceptance criteria in `USER_STORIES.md` are met
- [ ] PO has reviewed on device or simulator
- [ ] No known P0/P1 bugs for the story

## Engineering
- [ ] Code merged to `main` via PR
- [ ] At least one peer review approval
- [ ] Unit tests for business logic (where applicable)
- [ ] No new linter errors
- [ ] Environment variables documented in `.env.example`

## UX
- [ ] Loading and error states handled
- [ ] Matches agreed design or wireframe for the sprint

## Release
- [ ] Deployed to staging / internal build
- [ ] Story marked Done in sprint board
